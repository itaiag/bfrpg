## Project Overview
Quarto book site for Basic Fantasy RPG (BFRPG) rules. English source at root, Hebrew translation in `he/`. Built output goes to `docs/` (GitHub Pages).

## Translation System
All translator code lives in `translator/`. Key CLI commands:
```
node translator/translate.js <file.qmd>   # translate one or more files
node translator/translate.js --all        # translate all content files
node translator/translate.js --dry-run <file>  # preview segments, no API
node translator/translate.js --patch      # re-apply post-processors to he/ files
node translator/translate.js --build      # generate he/_quarto.yml + quarto render he/
node translator/translate.js --preview    # live preview of Hebrew site (hot-reload)
```

## Build Commands
```
quarto render          # English site â†’ docs/
quarto render he/      # Hebrew site â†’ docs/he/  (NOT --profile he)
```
**Important**: `--profile he` merges chapters additively with the root `_quarto.yml`. Always render `he/` as a standalone project instead.

## Architecture Decisions

### Hebrew Quarto Build
- `he/` is its own standalone Quarto project
- `he/_quarto.yml` is auto-generated by `--build` (gitignored)
- Uses `lang: he-IL` (not `lang: he` â€” conflicts with `he/` directory name)
- `output-dir: ../docs/he`

### Dice Notation
- Keep English `d` notation (1d6, 2d8, etc.) â€” do NOT use Hebrew ×§
- Reason: ×§ is RTL; bidi algorithm scrambles surrounding digits in Hebrew paragraphs
- Hebrew RPG publications standardly use English `d` notation

### Unit Conversion
- Imperial units are **replaced entirely** with metric (no parenthetical append)
- Game-friendly rounding: feetأ—0.3 (â†’ nearest 0.5m), milesأ—1.5 (â†’ nearest 0.5km), poundsأ—0.45 (â†’ nearest 1kg)
- Post-processor handles conversion; AI is told NOT to convert units
- Hebrew transliterations (×¤×™×ک, ×¨×’×œ, ×‍×™×™×œ, ×¤×گ×•× ×“) also converted

### AI Translation
- Default model: `gpt-4o` (configurable via `OPENAI_MODEL` env var)
- API key in `.env` (gitignored); template at `.env.example`
- Segments batched ~3000 tokens per call

### GM Gender
- Game Master (×‍× ×—×”) is always feminine in Hebrew

### Path Fixing
- All `he/*.qmd` files are one level deeper than root
- `fixRelativePaths()` handles: JS imports, FileAttachment, markdown `(./`, HTML src/href
- Applied to TRANSLATE, PRESERVE, and OJS segments
